# 用户版小程序后端上线说明

这个目录是用户版微信小程序的 HTTPS 后端适配层，负责：

- 使用 `wx.login` code 换取微信 `openid`；
- 用 `MP_ALLOWED_OPENIDS` 控制用户版白名单；
- 转发推荐和材料初稿请求到网页版 Render 服务；
- 默认开放 1/3/6 所推荐，对 10 所推荐、材料助手保留高级权限门控；
- 接收小程序预约提交，阻止同一老师同一日期同一时间重复预约，并按配置把预约通知同步给对应老师。

## Render 部署

1. 在 Render 新建 Web Service，根目录选择 `微信小程序/backend`。
2. Build Command 使用 `npm install`。
3. Start Command 使用 `npm start`。
4. 按 `.env.example` 配置环境变量。
5. 部署完成后访问 `/health`，看到 `ok: true` 即代表 adapter 已启动。

推荐服务名：

```text
hangzhou-zhihang-mp-adapter-zada
```

对应默认 HTTPS 地址：

```text
https://hangzhou-zhihang-mp-adapter-zada.onrender.com
```

如果 Render 实际生成了不同域名，需要同步修改用户版小程序 `utils/env.js` 里的 `API_BASE_URL`。

## 必填环境变量

- `WEB_ADVISOR_BASE_URL`：网页版 Render 地址，当前为 `https://hangzhou-zhihang-study-advisor-zada.onrender.com`。
- `WEB_ACCESS_PASSWORD`：网页版访问密码，必须和网页版 Render 的 `ACCESS_PASSWORD` 一致。
- `WECHAT_APPID`：用户版微信小程序 AppID。
- `WECHAT_SECRET`：用户版微信小程序 AppSecret。
- `MP_ALLOWED_OPENIDS`：允许登录用户版的 openid，英文逗号分隔。
- `MP_OPEN_LOGIN`：内部体验阶段可设为 `true`，让能扫码进入体验版的微信用户自动登录；正式公开发布前建议改回 `false` 或接入正式账号/支付权限。
- `MP_LOG_DENIED_OPENID`：临时设为 `true` 时，会在 Render 日志里记录未绑定登录用户的完整 openid，拿到首批 openid 后请改回 `false`。
- `MP_FREE_RECOMMENDATION_COUNTS`：用户版默认可选推荐数量，当前建议填 `1,3,6`。
- `MP_BOOKINGS_FILE`：预约占用记录文件，默认 `data/bookings.jsonl`。

## 可选环境变量

- `MP_USER_ENTITLEMENTS_JSON`：按 openid 配置高级权限，例如：

```json
{
  "openid_1": {
    "recommendationCount": true,
    "materialAssistant": true
  }
}
```

## 预约通知配置

当前后端支持两种通知方式，可以二选一，也可以同时配置：

Webhook 指的是“把预约消息发到一个固定的机器人地址”，通常用于企业微信群、飞书群或钉钉群。它不是个人微信服务通知。如果张老师和陆老师必须在个人微信里收到通知，请优先使用下面第 2 种微信小程序订阅消息。

1. 企业微信/飞书/钉钉机器人 Webhook：适合先把预约发到内部群。
   - 所有预约发到同一个群：设置 `MP_BOOKING_WEBHOOK_URL`。
   - 张老师、陆老师分别收不同通道：设置 `MP_BOOKING_TEACHER_WEBHOOKS_JSON`，例如：

```json
{
  "a1": "张老师群机器人 Webhook",
  "a2": "陆老师群机器人 Webhook"
}
```

2. 微信小程序订阅消息：老师会在微信服务通知里收到提醒。需要在微信公众平台配置模板，并让老师使用同一小程序授权接收该模板。
   - `MP_BOOKING_NOTIFY_ENABLED=true`
   - `MP_BOOKING_TEMPLATE_ID=订阅消息模板ID`
   - `MP_TEACHER_OPENIDS_JSON={"a1":["张老师openid"],"a2":["陆老师openid"]}`
   - 如果模板字段不是默认的 `thing1/time2/thing3/thing4`，用 `MP_BOOKING_TEMPLATE_FIELDS_JSON` 映射字段，例如：

```json
{
  "thing1": "studentName",
  "time2": "dateTime",
  "thing3": "advisorName",
  "thing4": "note"
}
```

老师授权步骤：

1. 微信公众平台添加“预约通知/服务预约提醒”订阅消息模板，复制模板 ID。
2. 在 Render 环境变量填入 `MP_BOOKING_TEMPLATE_ID`，并重新部署后端。
3. 张老师、陆老师分别登录小程序，进入“我的 - 设置 - 老师预约提醒授权”，点击“开启老师预约提醒”并允许订阅。
4. 获取两位老师的 openid，填入 `MP_TEACHER_OPENIDS_JSON`，例如 `{"a1":["张老师openid"],"a2":["陆老师openid"]}`。
5. Render 重新部署后，学生预约张老师会通知张老师，预约陆老师会通知陆老师。

如果以上通知变量没有配置，小程序仍会保存预约并给用户生成可复制的预约信息，不会阻断预约提交。

## 预约时间占用

小程序会按“老师 + 日期 + 时间”查询已占用时段，并在预约页置灰“已约”。后端提交时还会再次校验，如果同一时段已经被占用，会返回 409，前端提示用户重新选择。

当前实现按“提交即占用”处理，防止重复预约。若以后需要老师手动确认后才占用，可以继续扩展 `status=pending/confirmed/cancelled` 的后台管理。

重要：Render 免费实例的本地文件可能在重启或重新部署后丢失。为了让预约占用长期可靠，请给后端接入持久化存储：Render Disk、数据库，或把 `MP_BOOKINGS_FILE` 指向持久磁盘路径。

## 微信公众平台配置

在微信公众平台的小程序后台，进入开发管理里的服务器域名配置，把 adapter 域名加入 `request 合法域名`：

```text
https://hangzhou-zhihang-mp-adapter-zada.onrender.com
```

如果使用了不同 Render 域名，填写实际域名。
