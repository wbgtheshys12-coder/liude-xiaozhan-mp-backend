# 用户版小程序后端上线说明

这个目录是用户版微信小程序的 HTTPS 后端适配层，负责：

- 使用 `wx.login` code 换取微信 `openid`；
- 用 `MP_ALLOWED_OPENIDS` 控制用户版白名单；
- 使用小程序后端内置专业库生成院校专业推荐、成绩单识别预览和材料初稿，不再依赖网页版 Render 服务；
- 默认开放 1/3/6 所推荐，对 10 所推荐、材料助手保留高级权限门控；
- 接收小程序预约提交，阻止同一老师同一日期同一时间重复预约，并按配置把预约通知同步给对应老师。

## Render 部署

1. 在 Render 新建 Web Service，根目录选择 `微信小程序/backend`。
2. Build Command 使用 `npm install`。
3. Start Command 使用 `npm start`。
4. 按 `.env.example` 配置环境变量。
5. 部署完成后访问 `/health`，看到 `ok: true` 即代表 adapter 已启动。

推荐服务名：

```text
hangzhou-zhihang-mp-adapter-zada
```

对应默认 HTTPS 地址：

```text
https://hangzhou-zhihang-mp-adapter-zada.onrender.com
```

如果 Render 实际生成了不同域名，需要同步修改用户版小程序 `utils/env.js` 里的 `API_BASE_URL`。

## 必填环境变量

- `WECHAT_APPID`：用户版微信小程序 AppID。
- `WECHAT_SECRET`：用户版微信小程序 AppSecret。
- `OCR_ALLOW_REMOTE_TESSDATA`：是否允许 OCR 运行时远程下载语言包，国内部署建议保持 `false`。
- `OCR_TESSDATA_DIR`：可选本地 OCR 语言包目录；默认可为空，后端会优先使用 npm 安装的中英文语言包。
- `MP_ALLOWED_OPENIDS`：允许登录用户版的 openid，英文逗号分隔。
- `MP_OPEN_LOGIN`：内部体验阶段可设为 `true`，让能扫码进入体验版的微信用户自动登录；正式公开发布前建议改回 `false` 或接入正式账号/支付权限。
- `MP_LOG_DENIED_OPENID`：临时设为 `true` 时，会在 Render 日志里记录未绑定登录用户的完整 openid，拿到首批 openid 后请改回 `false`。
- `MP_FREE_RECOMMENDATION_COUNTS`：用户版默认可选推荐数量，当前建议填 `1,3,6`。
- `MP_BOOKINGS_FILE`：预约占用记录文件，默认 `data/bookings.jsonl`。

## 可选环境变量

- `MP_USER_ENTITLEMENTS_JSON`：按 openid 配置高级权限，例如：

```json
{
  "openid_1": {
    "recommendationCount": true,
    "materialAssistant": true
  }
}
```

## 老师管理员预约管理

本版本支持让负责人、张老师和陆老师登录小程序后，在“我的”页看到“预约管理”入口，进入后可查看全部预约记录、今日预约数、已占用时段，并可按老师筛选。

- `MP_TEACHER_OPENIDS_JSON`：推荐配置项。填入两位老师的 openid 后，两位老师会自动拥有预约管理权限，例如：

```json
{
  "a1": ["张老师openid"],
  "a2": ["陆老师openid"]
}
```

Render 环境变量的值需要直接填完整 JSON，不要在最外层再加一对引号，也不要漏掉 `{}`。公开文档、代码和压缩包中只能使用占位符，例如：

- 不需要成为预约老师、但需要完整内部管理权限的测试人员，应加入 `MP_ADMIN_OPENIDS_JSON=["测试人员openid"]`。该角色可以管理预约、课程、学生资料和统计，但不会成为学生可选择的顾问，也不会进入老师订阅消息名单。不要把同一个测试人员同时写入老师和管理员变量。
- 电脑后台地址是后端域名加 `/admin`。设置 `MP_ADMIN_WEB_TOKEN` 为单独生成的长随机 Secret 后启用；令牌可查看课程、预约、学生资料和统计，不要写入 GitHub、小程序源码、压缩包或聊天。
- 用户可在“我的 → 联系客服”发送消息；老师、负责人及 `MP_ADMIN_OPENIDS_JSON` 中的平台管理员可在小程序“客服消息”或电脑后台查看并回复。消息写入 `MP_DATA_DIR/messages.jsonl`，生产环境必须使用 Render Persistent Disk。
- `MP_DOCUMENT_DOWNLOAD_FREE=true` 表示上线初期动机信/CV 可免费生成和下载；以后收费能力完成后再改为 `false` 并配置用户权益。当前支付入口明确显示“待开发”。

```text
{"a1":["teacher_a1_openid"],"a2":["teacher_a2_openid"]}
```

如果误填成 `"a1":[...],"a2":[...]`，新版后端会尽量自动兼容，但仍建议改成上面的标准写法，避免 Render 日志持续出现解析警告。

- `MP_OWNER_OPENIDS_JSON`：负责人/老板本人使用。填入你的 openid 后，你也会拥有和老师一样的预约管理查看权限，例如：

```json
["你的openid"]
```

- `MP_ADMIN_OPENIDS_JSON`：可选补充项。如果还有其他内部人员也要看预约后台，可填：

```json
["内部人员openid1", "内部人员openid2"]
```

获取老师 openid 的临时方法：

1. 在 Render 环境变量里临时设置 `MP_LOG_LOGIN_OPENID=true`。
2. 让张老师、陆老师分别进入小程序并完成微信登录。
3. 打开 Render Logs，复制日志里输出的 openid。
4. 你的 openid 填入 `MP_OWNER_OPENIDS_JSON`；两位老师填入 `MP_TEACHER_OPENIDS_JSON`；其他内部人员填入 `MP_ADMIN_OPENIDS_JSON`。
5. 拿到 openid 后把 `MP_LOG_LOGIN_OPENID` 改回 `false`，避免日志继续暴露 openid。

## 预约通知配置

当前后端支持两种通知方式，可以二选一，也可以同时配置：

Webhook 指的是“把预约消息发到一个固定的机器人地址”，通常用于企业微信群、飞书群或钉钉群。它不是个人微信服务通知。如果张老师和陆老师必须在个人微信里收到通知，请优先使用下面第 2 种微信小程序订阅消息。

1. 企业微信/飞书/钉钉机器人 Webhook：适合先把预约发到内部群。
   - 所有预约发到同一个群：设置 `MP_BOOKING_WEBHOOK_URL`。
   - 同一个 Webhook 也会接收学生取消预约的提醒，包含学生、顾问、原预约时间、备注和预约编号。
   - 张老师、陆老师分别收不同通道：设置 `MP_BOOKING_TEACHER_WEBHOOKS_JSON`，例如：

```json
{
  "a1": "张老师群机器人 Webhook",
  "a2": "陆老师群机器人 Webhook"
}
```

2. 微信小程序订阅消息：老师会在微信服务通知里收到提醒。需要在微信公众平台配置模板，并让老师使用同一小程序授权接收该模板。
   - `MP_BOOKING_NOTIFY_ENABLED=true`
   - `MP_BOOKING_TEMPLATE_ID=订阅消息模板ID`
   - `MP_TEACHER_OPENIDS_JSON={"a1":["张老师openid"],"a2":["陆老师openid"]}`
   - 必须按微信公众平台模板详情显示的真实字段编号配置 `MP_BOOKING_TEMPLATE_FIELDS_JSON`。“上课通知”模板常见映射示例：

```json
{
  "time1": "dateTime",
  "thing2": "留学咨询预约",
  "thing3": "studentName",
  "thing4": "advisorName",
  "thing5": "note"
}
```

老师授权步骤：

1. 微信公众平台添加“预约通知/服务预约提醒”订阅消息模板，复制模板 ID。
2. 在 Render 环境变量填入 `MP_BOOKING_TEMPLATE_ID`，并重新部署后端。
3. 张老师、陆老师分别登录小程序，进入“我的 - 设置”或“预约管理”，点击“补充一次预约提醒授权”并允许订阅。
4. 获取两位老师的 openid，填入 `MP_TEACHER_OPENIDS_JSON`，例如 `{"a1":["张老师openid"],"a2":["陆老师openid"]}`。
5. Render 重新部署后，学生预约任意老师都会同步通知两位已配置老师，避免合伙人漏看预约。小程序内“预约管理”仍可按张老师/陆老师筛选。

注意：微信订阅消息必须由接收人本人在小程序内点击按钮授权，不能由后台静默替老师授权。一次性订阅通常在成功发送一条通知后消耗；老师每收到一条预约通知后，应再次进入“我的 - 设置”或“预约管理”补充下一次授权。体验版使用 `MP_BOOKING_MINIPROGRAM_STATE=trial`，正式发布后再改为 `formal`。

如微信公众平台中的模板详情明确标注为“长期订阅”，可增加 `MP_BOOKING_SUBSCRIPTION_MODE=long-term`。两位老师各自允许一次后，小程序会按长期提醒展示，不再引导反复补充授权。若模板仍是“一次性订阅”，不能仅靠修改代码或环境变量获得永久发送权限；应继续使用 `one-time`，或配置 `MP_BOOKING_WEBHOOK_URL` 将每条预约持续推送到两位老师所在的企业群。不要把一次性模板伪装成长期模板，否则后续通知仍会被微信拒绝。

如果以上通知变量没有配置，小程序仍会保存预约并给用户生成可复制的预约信息，不会阻断预约提交。

## 预约时间占用

小程序会按“老师 + 日期 + 时间”查询已占用时段，并在预约页置灰“已约”。后端提交时还会再次校验，如果同一时段已经被占用，会返回 409，前端提示用户重新选择。

当前实现按“提交即占用”处理，防止重复预约。若以后需要老师手动确认后才占用，可以继续扩展 `status=pending/confirmed/cancelled` 的后台管理。

重要：Render 免费实例的本地文件可能在重启或重新部署后丢失。预约、课程、学生资料和使用统计必须使用持久化存储。可配置 Render Disk 并把 `MP_DATA_DIR` 指向挂载目录；部署后 `/health` 的 `externalPersistentDataDirConfigured` 应为 `true`。仅显示 `studentUploadDatabaseEnabled=true` 不代表磁盘已经持久化。如果 Render 里已经存在 `MP_BOOKINGS_FILE`、`MP_USAGE_FILE`、`MP_COURSE_VIDEO_DIR`、`MP_TRANSCRIPT_TEMPLATES_FILE` 等逐文件变量，要删除它们或全部改到同一持久磁盘目录，否则它们会覆盖 `MP_DATA_DIR`。

## 网课管理与使用统计

老师或负责人 openid 配置完成后，登录小程序进入“我的 - 内部管理”可以看到：

- “课程管理”：电脑后台与小程序后台使用同一个课程数据源，刷新后同步显示。可以新增、编辑、删除录播课和直播课；上传后会显示文件名、大小和文件是否存在，编辑课程可回显稳定的视频记录；移除课程视频会同时清理本后端保存的文件。录播课也可填写 HTTPS 视频地址，直播课填写腾讯会议、企业微信直播、腾讯云直播等观看入口。
- “学生资料库”：查看并下载/保存学生在小程序上传的申请材料。学生本人也可查看、保存和删除自己的文件。
- “使用统计”：查看预约总数、今日预约、资料上传、课程数量、活跃用户、最近使用行为，并可复制导出完整后台数据。

课程视频相关环境变量：

- `MP_COURSE_VIDEO_DIR`：本地课程视频保存目录，默认 `data/course-videos`。
- `MP_MAX_COURSE_VIDEO_BYTES`：单个课程视频上传上限，默认约 35MB。
- `MP_MEDIA_SIGNING_SECRET`：课程视频短时签名密钥，只能放在 Render Secret 环境变量中。
- `MP_MEDIA_URL_TTL_SECONDS`：本地课程视频签名地址有效期，默认 3600 秒。
- `MP_SESSION_TTL_SECONDS`：小程序登录令牌有效期，默认 2592000 秒（30 天）。
- `MP_SESSION_SECRET`：可选的登录令牌加密密钥；未配置时自动复用 `MP_MEDIA_SIGNING_SECRET`。生产环境必须保证其中至少一个稳定密钥存在，不能在每次部署时变化。

登录令牌使用带完整性校验的加密格式，不再依赖 Render 进程内存。因此服务休眠、重启或重新部署后，尚未过期的登录状态仍可继续使用；小程序遇到旧版令牌或过期令牌时会自动调用微信登录并重试当前请求，不会反复要求用户手动登录。部署后可在 `/health` 确认 `sessionTokensStateless=true`、`sessionSigningStable=true` 和 `sessionTokenTtlSeconds=2592000`。

安全要求：生产环境不要配置 `MP_ALLOW_DEV_LOGIN=true`。`/api/mp/demo/login` 仅在该变量明确开启时可用，线上应保持关闭，确保课程访问必须经过真实微信登录。

学生首次登录需要一次性填写姓名、联系方式、当前/毕业学校、当前/本科专业和申请层次。资料绑定微信账号并锁定，预约和客服均由后端读取这份档案，客户端重复提交或伪造身份字段不会覆盖档案。旧版本中已经锁定但字段不全的档案允许仅补充缺失字段，原有非空字段不能被修改。

客服消息使用连续对话形式，学生与老师均会每 15 秒自动刷新。学生发送新消息后，默认复用 `MP_BOOKING_WEBHOOK_URL` / `MP_BOOKING_TEACHER_WEBHOOKS_JSON` 通知企业微信群；提醒包含学生姓名、联系方式、学校、专业和申请层次，便于老师识别会话，但不包含咨询原文、openid 或文件内容。如需让客服消息进入单独的企业群，可另设 `MP_CUSTOMER_MESSAGE_WEBHOOK_URL`。

注意：本地上传的视频使用绑定微信账号的短时签名地址，并支持 Range 播放；页面不提供下载按钮。微信无法绝对阻止手机系统录屏。Render 免费实例本地磁盘不适合长期保存大视频。正式网课建议使用带防盗链/签名和账号鉴权的腾讯云点播等服务；普通外部 HTTPS 直链本身可能被转发，不能等同于完整 DRM。

学生成绩单只有在本人开启“去标识化成绩单识别改进授权”后，才标记为 OCR 改进素材。私有模板放在 `MP_TRANSCRIPT_TEMPLATES_FILE` 指向的持久化私有文件中，不能提交到 GitHub、发布包或公开数据集；模板只按文件 SHA-256 精确匹配，避免把一个学生的课程/成绩误套到另一个学生。

## 文书导出

用户版小程序“核心服务 - 动机信/简历/课程描述”支持 Word 和 PDF 导出：

- “导出调查表 Word”：学生填写后即可导出带留德小栈水印的调查表，方便发给顾问核对。
- “导出完整文书 Word”：`MP_DOCUMENT_DOWNLOAD_FREE=true` 时上线初期免费下载；以后关闭免费开关后恢复权益校验和前 20% 预览。
- “导出调查表 PDF（水印）”：生成带留德小栈水印的 PDF，可在微信文档菜单中保存。
- “导出文书 PDF（水印）”：免费期导出全文水印版；以后关闭免费开关后，未开通用户只导出前 20% 水印预览。

导出的 Word 为小程序本地生成的 `.doc` 水印文件，不会把学生资料发送到第三方文书服务。

## 微信公众平台配置

在微信公众平台的小程序后台，进入开发管理里的服务器域名配置，把 adapter 域名加入 `request 合法域名`：

```text
https://hangzhou-zhihang-mp-adapter-zada.onrender.com
```

如果使用了不同 Render 域名，填写实际域名。
